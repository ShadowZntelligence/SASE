===============================================================================
WHAT IS DOMAIN KNOWLEDGE?
===============================================================================

Domain knowledge describes semantic properties of valid program behavior rather
than implementation details.

Examples include:

- protocol semantics

- parser semantics

- file format semantics

- valid message sequences

- legal state transitions

- feature activation conditions

- semantic constraints

- relationships among input elements

===============================================================================

----------------------------------------------------------------------------
2. Semantic Constraints
----------------------------------------------------------------------------

Identify semantic relationships.

Examples

MOOV should appear before media playback.

HTTP header must be parsed before body.

TLS handshake must complete before application data.

Authentication must succeed before resource access.

Do NOT include syntax constraints.

Focus on semantic constraints.

----------------------------------------------------------------------------
3. Feature Activation Conditions
----------------------------------------------------------------------------

Identify inputs that activate different functionalities.

Examples

Specific box type

Specific protocol version

Specific command

Specific message type

Specific parser mode

Specific feature flag

Specific configuration

----------------------------------------------------------------------------
4. State Machine Transitions
----------------------------------------------------------------------------

Identify meaningful semantic state transitions.

Examples

INIT

↓

HEADER_PARSED

↓

BODY_PARSED

↓

REQUEST_HANDLED

or

WAIT_CLIENT_HELLO

↓

NEGOTIATING

↓

ESTABLISHED

Avoid implementation states.

----------------------------------------------------------------------------
5. Behavioral Dependencies
----------------------------------------------------------------------------

Identify dependencies between semantic behaviors.

Examples

Metadata parsing before media parsing.

Authentication before authorization.

Container validation before decoding.

Request parsing before dispatch.

----------------------------------------------------------------------------
6. Security-Relevant Constraints
----------------------------------------------------------------------------

Identify security-related semantic knowledge.

Examples

Authentication requirements.

Permission checks.

Input validation requirements.

Boundary constraints.

Length consistency.

Cross-field consistency.

Resource ownership.

Privilege transitions.

Cryptographic verification.

----------------------------------------------------------------------------
7. Exceptional Behaviors
----------------------------------------------------------------------------

Identify abnormal but meaningful behaviors.

Examples

Unsupported protocol version.

Malformed file.

Invalid state transition.

Unexpected message order.

Incomplete header.

Invalid metadata.

Corrupted structure.
