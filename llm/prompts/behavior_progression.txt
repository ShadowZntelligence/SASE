===============================================================================
GOAL
===============================================================================

Infer multiple meaningful semantic evolution paths.

Each progression should represent one possible behavioral evolution of the
target function under the current execution context.

Think from the perspective of:

"What meaningful semantic behaviors should symbolic execution try to reach?"

rather than

"What branches should symbolic execution execute?"

===============================================================================
WHAT IS A BEHAVIORAL PROGRESSION?
===============================================================================

A behavioral progression is a sequence of semantic behaviors.

It describes how execution semantically evolves.

GOOD examples

Process FTYP
↓

Process MOOV
↓

Process MDAT

--------------------------------------------

Authenticate User
↓

Establish Session
↓

Execute Command

--------------------------------------------

Receive Request
↓

Parse Header
↓

Validate Request
↓

Dispatch Handler

--------------------------------------------

Load ELF Header
↓

Validate Program Header
↓

Load Segments

BAD examples

Execute branch A

Call parse_box()

Loop exits

Variable x becomes 1

Compare y > z

Invoke memcpy()

These are implementation details.

===============================================================================
INFER HIGH-VALUE BEHAVIORAL PATHS
===============================================================================

Infer behaviors that are likely to expose different semantic regions.

Examples include

- different protocol states

- different feature activation paths
