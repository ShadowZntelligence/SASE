===============================================================================
SEMANTIC TRACE DEFINITION
===============================================================================

A semantic trace is an ordered sequence of semantic tags.

Formally:

ST = <τ1, τ2, ..., τn>

where each τi is a semantic tag generated during Semantic Tag Creation.

A semantic trace represents a meaningful semantic evolution path.

A semantic trace does NOT represent:

- CFG paths
- execution paths
- branch decisions
- API call sequences
- source line sequences

Instead it represents:

- semantic progression
- functionality activation
- protocol evolution
- parser evolution
- feature activation

===============================================================================
IMPORTANT CONSTRAINT
===============================================================================

You MUST ONLY use semantic tags provided in the input.

Do NOT invent new semantic tags.

Do NOT rename semantic tags.

Do NOT modify semantic tags.

Every trace element must come from the provided semantic tag set.

===============================================================================
SEMANTIC TRACE CONSTRUCTION
===============================================================================

Given:

- semantic profile
- ranked objectives
- semantic tags

Construct semantic traces that represent the semantic progression
required to achieve each objective.

FTYP_PROCESSING
MOOV_PROCESSING
MDAT_PROCESSING
MOOF_PROCESSING

  "TOP_LEVEL_BOX_DISCOVERED",
  "MOOV_PROCESSING",
  "TOP_LEVEL_BOX_DISCOVERED",
